# 情感表白网站

一个纯前端单页网站，用于通过文字+音乐的方式向爱人表达心意。

## 功能特点

- **点击触发文字显示**：点击屏幕依次显示每一段文字，当前段落淡出后下一段淡入
- **背景音乐控制**：右下角常驻控制按钮，支持静音/取消静音、重新开始
- **段落联动音乐**：不同段落可配置不同的背景音乐，切换时自动淡入淡出
- **响应式设计**：适配手机和电脑浏览器
- **循环播放**：音乐默认循环播放，直到切换到新音乐

## 配置方式

打开 `index.html` 文件，找到「配置区」修改以下内容：

### 文字内容配置

```javascript
const STORY = [
    {
        text: "亲爱的，<strong>对不起</strong>……",
        music: "music/music1.mp3"
    },
    {
        text: "还记得那天……",
        music: "music/music2.mp3"
    },
    {
        text: "以后我会……",
        music: ""  // 空字符串表示延续上一首音乐
    }
];
```

### 其他配置项

```javascript
const CROSS_FADE_DURATION = 1500;     // 音乐切换淡入淡出时间（毫秒）
const DEFAULT_VOLUME = 0.3;           // 默认音量（0-1）

const ENDING_TEXT = "永远爱你的";      // 结尾前缀
const ENDING_SIGNATURE = "XXX";       // 签名
```

### 文字样式

- 使用 `<strong>文字</strong>` 可使文字变红加粗
- 使用 `<br>` 可换行

## 音乐文件

将你的 MP3/WAV 音乐文件放入 `music` 文件夹，然后在配置区引用。

示例：
```
music/
├── music1.mp3
├── music2.mp3
└── music3.mp3
```

## 交互说明

1. **开场**：点击屏幕任意位置开始故事，同时播放第一段背景音乐
2. **阅读**：每次点击屏幕，当前段落淡出，下一段淡入显示
3. **音乐**：当显示到有音乐配置的段落时，自动切换到对应的音乐（淡入淡出过渡）
4. **结束**：所有段落读完后，显示签名和爱心
5. **重新开始**：点击结束画面或右下角重新开始按钮，从头播放

## 部署方式

### 方式一：Vercel（推荐）

1. 将项目代码推送到 GitHub 仓库
2. 访问 [Vercel](https://vercel.com/)，登录后点击「New Project」
3. 选择你的 GitHub 仓库，点击「Deploy」
4. 部署完成后会获得一个可访问的 URL

### 方式二：GitHub Pages

1. 将项目代码推送到 GitHub 仓库
2. 进入仓库设置页面，找到「Pages」选项
3. 在「Source」中选择 `main` 分支和 `/root` 文件夹
4. 点击「Save」，等待几分钟后会获得一个可访问的 URL

### 方式三：本地预览

```bash
# 使用 Python（推荐）
python -m http.server 8080

# 或使用 http-server
npx http-server -p 8080
```

然后在浏览器中访问 `http://localhost:8080`

## 注意事项

1. **浏览器自动播放限制**：页面加载后不会自动播放音乐，需要用户首次点击屏幕后才会开始播放
2. **音乐文件格式**：支持 MP3、WAV 等常见格式
3. **音乐版权**：请确保你拥有所使用音乐的版权
4. **移动端兼容性**：在 Safari 等严格音频限制的浏览器上，需要用户交互后才能播放音乐

## 快捷键

- **空格键**：开始故事 / 显示下一段 / 重新开始
- **M 键**：静音/取消静音
